# Writing Rules - Phạm Thành Nam's Blog Style

## Tone Characteristics

### Core Principles
- **Gần gũi**: Viết như đang nói chuyện với đồng nghiệp, không quá formal
- **Rõ ràng**: Đi thẳng vào vấn đề, không vòng vo
- **Không màu mè**: Tránh những từ ngữ hoa mỹ, marketing-style
- **Tư duy kỹ sư**: Giải thích "vì sao" trước khi nói "làm như thế nào"

### Writing Approach
1. Bắt đầu với context và vấn đề thực tế
2. Giải thích lý do đằng sau giải pháp
3. Đưa ra ví dụ cụ thể, code thực tế
4. Kết thúc với action items rõ ràng

## Target Audience

### Primary Audience
- **Developer**: Người có kinh nghiệm lập trình, hiểu technical concepts
- **Technical Founder**: Người làm startup có background kỹ thuật

### Audience Expectations
- Muốn hiểu bản chất vấn đề, không chỉ copy-paste solution
- Đánh giá cao practical examples hơn lý thuyết suông
- Thích đọc những bài ngắn gọn, đi thẳng vào vấn đề

## Language Rules

### Primary Language
- **Vietnamese** là ngôn ngữ chính cho nội dung
- Tỷ lệ Vietnamese characters: >80%

### English Technical Terms
Sử dụng tiếng Anh cho các thuật ngữ kỹ thuật phổ biến:
- API, REST, GraphQL, WebSocket
- Database, Cache, Queue
- Frontend, Backend, Fullstack
- Deploy, CI/CD, Docker, Kubernetes
- Framework names: React, Vue, Node.js, etc.

### Language Guidelines
- Không dịch thuật ngữ kỹ thuật sang tiếng Việt nếu không cần thiết
- Giữ nguyên tên công nghệ, library, framework
- Có thể dùng tiếng Anh cho code comments

## Formatting Rules

### Markdown Structure
- Sử dụng Markdown format cho tất cả output
- Heading hierarchy: H1 → H2 → H3 (không skip level)

### Heading Guidelines
```
# Title (H1) - Chỉ dùng 1 lần cho tiêu đề bài
## Section (H2) - Các phần chính của bài
### Subsection (H3) - Chi tiết trong mỗi phần
```

### Paragraph Rules
- **Maximum 5 lines per paragraph**
- Mỗi paragraph nên focus vào 1 ý chính
- Dùng line break để tách các ý khác nhau

### List Usage
- Dùng bullet list cho các items không có thứ tự
- Dùng numbered list cho steps hoặc items có sequence
- Không lạm dụng list - chỉ dùng khi cần thiết

### Code Blocks
- Luôn specify language cho syntax highlighting
- Ví dụ: ```javascript, ```python, ```typescript
- Code snippets nên ngắn gọn, focus vào điểm chính

## Sentence Guidelines

### Length
- **Short to medium sentences**: 5-50 words per sentence
- Tránh câu quá dài, phức tạp
- Chia câu dài thành nhiều câu ngắn hơn

### Structure
- Subject-Verb-Object rõ ràng
- Tránh passive voice khi có thể
- Dùng active voice để tăng clarity

## Content Priorities

### What to Include
1. Real-world problems và context
2. Reasoning behind solutions
3. Practical code examples
4. Common mistakes to avoid
5. Actionable takeaways

### What to Avoid
1. Marketing phrases (xem blacklist)
2. Abstract explanations without examples
3. Off-topic tangents
4. Repetitive content
5. Overly long introductions
